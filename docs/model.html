Pourquoi "lite" ? Quatre convolutions simples sans connexions résiduelles — entraînement rapide, idéal pour valider le pipeline complet en développement. Sortie : un vecteur global (pas de grille spatiale), donc pas de mécanisme d'attention possible. Aucun pré-entraîné.

✅ Avantages

  • Très rapide à entraîner
  • Peu de paramètres
  • Facile à déboguer

⚠️ Limites

  • Performances basses sur COCO
  • Pas d'attention visuelle
  • Gradient flow faible (pas de résidus)

Quand l'utiliser ?

Phase de développement, tests du pipeline, machines sans GPU. Jamais en production.

Différence avec Lite : La connexion résiduelle (x + f(x)) permet au gradient de "court-circuiter" les couches pendant la rétropropagation. Plus de profondeur sans dégradation du gradient. Mais comme EncoderLite, la sortie reste un vecteur global : aucune information spatiale n'est transmise au decoder. EncoderSpatial (panel "attention") corrige ce problème.

Différence clé : vecteur global vs grille spatiale

❌ EncoderCNN (full) — vecteur global

AdaptiveAvgPool(1×1) efface toute la dimension spatiale. Le decoder reçoit un seul vecteur résumant toute l'image. Il ne peut pas "regarder" une région spécifique pour écrire un mot.

✅ EncoderSpatial (attention) — grille 7×7

AdaptiveAvgPool(7×7) conserve 49 "patches". Chaque patch correspond à une région ~32×32 pixels de l'image originale. Le decoder peut choisir quelle région regarder à chaque mot généré.

Le mécanisme d'attention de Bahdanau — pas à pas

1
Projection des features encodeur

Pour chacun des 49 patches de l'image : W_enc · f_i → projection dans l'espace d'attention (attention_dim=256). Chaque patch est projeté séparément.

2
Projection de l'état caché courant

L'état caché du LSTM au temps t : W_dec · h_t → même espace d'attention. Représente "ce que le decoder cherche" à cet instant.

3
Score d'énergie pour chaque patch

Pour chaque patch i : energy_i = v · tanh(W_enc·f_i + W_dec·h_t). Un score haut signifie "ce patch est pertinent pour le prochain mot".

4
Softmax → poids d'attention α

α_i = softmax(energy_i). Les 49 poids somment à 1 : c'est une distribution de probabilité sur les régions de l'image. Ces poids sont les alphas visualisables.

5
Vecteur de contexte

context = Σᵢ α_i · f_i : somme pondérée des 49 patches. C'est le "résumé visuel" centré sur la région importante pour générer le mot courant.

6
LSTMCell : contexte + embedding → prochain mot

h_t, c_t = LSTMCell(concat[emb(token_t), context], (h_{t-1}, c_{t-1})). Le LSTMCell reçoit à la fois le mot précédent (via embedding) et la région pertinente de l'image.

7
Régularisation Doubly Stochastic (entraînement)

Pénalité : λ · mean((1 - Σ_t α[:,t,p])²). Force le modèle à "regarder" chaque patch au moins une fois sur l'ensemble de la séquence — évite que l'attention se focalise toujours sur les mêmes coins.

🎯 Intuition : Pour "a dog runs on the grass", le modèle génère "dog" en regardant la région du chien (α élevés sur ce patch), puis "grass" en déplaçant l'attention vers le bas de l'image. Les alphas permettent de visualiser ce que regarde le modèle mot par mot.

✅ Avantages

  • Captions bien plus précises (CIDEr+)
  • Attention interprétable/visualisable
  • Régularisation doubly stochastic

⚠️ Limites vs DenseNet

  • Features résiduelles moins riches que DenseNet
  • Pas de réutilisation multi-échelle
  • BLEU-4 légèrement inférieur

API spécifique

forward_with_alphas() : retourne (outputs, alphas B×T×49).
generate_caption_with_attention() : génère + alphas visualisables.

🔑 La connexion dense — ce qui rend DenseNet unique

ResNet : chaque couche reçoit uniquement la sortie de la couche précédente : x_L = H_L(x_{L-1})

DenseNet : chaque couche reçoit la concaténation de TOUTES les couches précédentes du bloc : x_L = H_L([x_0, x_1, ..., x_{L-1}])

→ Les features de bas niveau (bords, textures) restent directement accessibles aux couches profondes. Pour le captioning, l'attention peut exploiter simultanément des features haute et basse résolution.

Couche de Transition — rôle du paramètre θ (compression)

θ
BN → ReLU → Conv1×1 (compression) → AvgPool2×2

θ=0.5 réduit les canaux de moitié avant le downsampling spatial. Double rôle : (1) comprime l'information — force le réseau à sélectionner les features importantes, (2) réduit le coût mémoire et calcul des blocs suivants. Sans compression (θ=1), les canaux exploseraient vers 2000+ dès le bloc 3.

Pourquoi DenseNet est meilleur pour le captioning avec attention ?

🌊
Gradient flow optimal

Les connexions denses créent des chemins courts depuis la loss jusqu'à chaque couche — pas de vanishing gradient même profond, entraînement from scratch facilité.

🔬
Multi-échelle native

La grille 7×7 finale contient des features issues de toutes les profondeurs : bords (couches 1), textures (couches 2-3), formes (couches 4+). L'attention peut les exploiter toutes.

♻️
Réutilisation des features

Chaque couche accède à toutes les couches précédentes → moins de paramètres nécessaires pour une profondeur et une richesse équivalentes à un ResNet.

🎯
Attention plus précise

Pour générer "airplane", l'attention peut s'appuyer sur des features de forme (profondes) ; pour "runway", sur des features de texture (peu profondes). Les deux coexistent dans la grille.

DenseNet-121 vs DenseNet-169 : block_config=(6,12,24,16) → DenseNet-121 (défaut, recommandé). block_config=(6,12,32,32) → DenseNet-169, plus lourd (~35% de paramètres en plus), légère amélioration sur COCO selon les configs. Le growth_rate=32 est le paramètre standard de l'article original (Huang et al. 2017).

Résiduel vs Dense — la différence visuelle

🟣 ResNet — addition (+)

Chaque couche reçoit la sortie de la précédente. La connexion résiduelle ajoute l'entrée à la sortie (x + f(x)). Le gradient peut passer directement mais les features ne s'accumulent pas.


x₁ = H₁(x₀) + x₀
x₂ = H₂(x₁) + x₁
x₃ = H₃(x₂) + x₂

🟠 DenseNet — concaténation (cat)

Chaque couche reçoit la concaténation de toutes les sorties précédentes. Les features s'accumulent. La couche 4 "voit" encore les features brutes de la couche 1.


x₁ = H₁([x₀])
x₂ = H₂([x₀, x₁])
x₃ = H₃([x₀, x₁, x₂])

Recommandation

🚀
lite

Vous commencez, vous testez le pipeline, vous n'avez pas de GPU. Durée : quelques minutes.

📐
full

Vous voulez une baseline propre avec résidus, sans la complexité de l'attention. Bon point de comparaison.

👁️
attention

Vous voulez visualiser l'attention, production légère, bon BLEU. Baseline forte pour publications.

⭐
densenet (recommandé COCO)

Meilleur BLEU-4 et CIDEr, entraînement from scratch solide, features multi-échelle + attention Bahdanau. Choix par défaut.
