EncoderCNN (version complète)

  • 5 blocs convolutionnels inspirés de VGG
  • Channels : 3 → 64 → 128 → 256 → 512 → 512
  • Initialisation Kaiming (adapté au ReLU)
  • ~14 millions de paramètres

EncoderCNNLite (version légère)

  • 4 blocs seulement, channels réduits
  • 3 → 32 → 64 → 128 → 256
  • Entraînement plus rapide
  • Recommandé pour tester / prototyper

🔤 decoder.py — Génération de texte mot par mot

Le DecoderLSTM reçoit le vecteur de features et génère une suite de mots formant la caption.

1

Initialisation du LSTM

Le vecteur de features (512 dim) est projeté via feature_projection pour initialiser le hidden state h₀ du LSTM.

2

Token <START>

La génération commence avec le token spécial <START>. Il est converti en embedding (256 dim) via la couche d'embedding.

3

Boucle LSTM

À chaque pas, l'embedding du mot courant entre dans le LSTM. Le hidden state (512 dim) est mis à jour.

4

Prédiction du prochain mot

La couche FC projette le hidden state sur tout le vocabulaire (ex: 5 000 mots). Le mot avec le score le plus haut est sélectionné (greedy search).

5

Arrêt

La boucle s'arrête quand le token <END> est généré, ou quand la longueur max est atteinte.

Mode entraînement — Teacher Forcing

Pendant l'entraînement, les vrais mots de la caption sont donnés comme entrée à chaque pas (même si le LSTM se trompe). Cela accélère et stabilise l'apprentissage.

Mode inférence — Greedy / Beam

Greedy : prend toujours le mot le plus probable. Rapide mais sous-optimal.
Beam Search : explore plusieurs séquences en parallèle pour de meilleurs résultats (non encore implémenté, redirige vers greedy).

📦 caption_model.py — Orchestration complète

Ce fichier colle tout ensemble et expose les fonctions de haut niveau.

ENTRAÎNEMENT

forward(images, captions)

  • Passe les images dans l'encoder → features
  • Passe les features + captions dans le decoder
  • Retourne les logits pour calculer la loss
INFÉRENCE

generate_caption(image)

  • Mode eval() + no_grad()
  • Supporte 'greedy' et 'beam_search'
  • Gère l'image 3D ou 4D (ajoute batch dim)
UTILITAIRE

save_model() / load_model()

  • Sauvegarde état encoder + decoder + config
  • Stocke optionnellement optimizer, epoch, loss, vocab
  • Recrée l'architecture au chargement
FACTORY

create_model()

  • Point d'entrée unique pour créer le modèle
  • Choisit encoder 'full' ou 'lite'
  • Configure tous les hyperparamètres

Hyperparamètres clés

vocab_size

Nombre de mots du vocabulaire (ex: 5 000). Définit la taille de la couche de sortie du decoder.

embedding_dim

Dimension des vecteurs de mots (default: 256). Chaque mot est représenté par un vecteur de cette taille.

hidden_dim

Taille du hidden state du LSTM (default: 512). Plus grand = plus de capacité mémoire.

feature_dim

Taille du vecteur de features de l'encoder (default: 512). Doit être identique entre encoder et decoder.
