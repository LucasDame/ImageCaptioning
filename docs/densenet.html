🔑 L'idée centrale

Dans un réseau classique, l'information "disparaît" dans les couches profondes. DenseNet résout ça : chaque couche voit toutes les couches précédentes. La couche 20 voit encore les features brutes de la couche 1.

🎯 Pourquoi pour le captioning ?

La grille 7×7 finale contient des features à toutes les échelles : textures basse fréquence ET formes complexes. L'attention Bahdanau peut exploiter les deux pour choisir les bons mots.

② Le Stem — entrée du réseau

Réduction rapide de la résolution : 224×224 → 56×56, 3 canaux → 64 canaux

opérations du stem — couche par couche
1

Conv2d(3→64, kernel=7×7, stride=2, padding=3)

Une grosse convolution 7×7 qui divise la résolution par 2 (stride=2). Elle transforme les 3 canaux RGB en 64 feature maps. Le padding=3 garantit que la formule (224 + 2×3 - 7) / 2 + 1 = 112 donne bien 112×112.

Entrée : (B, 3, 224, 224)
Sortie : (B, 64, 112, 112)
Paramètres : 64 × 3 × 7 × 7 = 9 408
Pourquoi 7×7 et pas 3×3 ? Une grande convolution en entrée capture des patterns à large réceptive field dès le début (bords, gradients de couleur), sans encore doubler la résolution.
2

BatchNorm2d(64)

Normalise chaque canal indépendamment sur le batch et les dimensions spatiales. Pour chaque canal c :

μ_c = moyenne sur (B, H, W)
σ_c = écart-type sur (B, H, W)
x̂ = (x - μ_c) / √(σ_c² + ε)
sortie = γ · x̂ + β ← γ, β appris

Accélère l'entraînement, stabilise les gradients, réduit la dépendance à l'initialisation.

3

ReLU(inplace=True)

Non-linéarité simple : f(x) = max(0, x). Les activations négatives passent à 0. inplace=True économise de la mémoire en modifiant le tenseur en place.

Sortie : (B, 64, 112, 112) — même shape
Paramètres : 0 (pas de poids appris)
4

MaxPool2d(kernel=3×3, stride=2, padding=1)

Prend le maximum dans chaque fenêtre 3×3, avec stride=2 → encore ÷2 en résolution. Le max-pool est agressif : il garde les features les plus saillantes et supprime le bruit.

Entrée : (B, 64, 112, 112)
Sortie : (B, 64, 56, 56) ← après le stem complet
Réduction totale : 224 ÷ 4 = 56
En résumé : le Stem réduit brutalement la résolution ×4 (de 224 à 56) tout en enrichissant les canaux (3→64). C'est un "zooming out" rapide qui rend les blocs suivants moins coûteux en calcul.

chaque opération expliquée
BN

BatchNorm2d — avant la conv (DenseNet inverse l'ordre !)

Dans un CNN classique : Conv → BN → ReLU. Dans DenseNet : BN → ReLU → Conv (pré-activation). Pourquoi ? Parce que l'entrée est une concaténation de sorties de couches précédentes qui ont déjà leur propre normalisation. En normalisant en premier, on évite que les features d'une couche ancienne "dominent" celles d'une couche récente.

Entrée : (B, C_in, H, W) ← C_in = 64 + i × 32 pour la couche i
Sortie : (B, C_in, H, W) ← même shape, normalisé par canal
ReLU

ReLU — non-linéarité

f(x) = max(0, x). Passe à 0 toutes les activations négatives. Sans ReLU, empiler des couches linéaires ne sert à rien (tout reste linéaire).

1×1

Conv2d(C_in → 4k, kernel=1×1) — le Bottleneck

C'est l'astuce clé qui évite l'explosion du nombre de paramètres. Sans elle, la Conv3×3 recevrait des centaines de canaux (et ses paramètres exploseraient). Avec 4k = 4×32 = 128 canaux, on comprime avant la conv coûteuse.

Exemple couche 10 du bloc 1 :
C_in = 64 + 9×32 = 352 canaux
→ Conv1×1 comprime vers 128 (= 4×32)
Paramètres sans bottleneck : 352 × 32 × 3 × 3 = 101 376
Paramètres avec bottleneck : 352×128×1 + 128×32×9 = 81 920 ← -20%
Un Conv1×1 est une transformation linéaire sur les canaux uniquement — aucun "regard" spatial. Il fait une sélection/combinaison des features existantes.
BN

BatchNorm2d(4k=128) — 2ᵉ normalisation

Normalise encore après le bottleneck, avant la Conv3×3. Les features compressées peuvent avoir des distributions très différentes selon les canaux — le BN recentre tout.

ReLU

ReLU — 2ᵉ non-linéarité

Deuxième activation avant la conv principale. Chaque "BN→ReLU→Conv" est un sous-bloc de transformation non-linéaire.

3×3

Conv2d(4k → k, kernel=3×3, padding=1) — la conv principale

La conv 3×3 avec padding=1 conserve la résolution spatiale (H et W inchangés). Elle produit exactement k = growth_rate = 32 nouveaux canaux — c'est la "contribution" de cette couche.

Entrée : (B, 128, H, W)
Sortie : (B, 32, H, W) ← k = growth_rate nouveaux canaux
Paramètres : 128 × 32 × 3 × 3 = 36 864
cat

torch.cat([x, out], dim=1) — La connexion dense !

C'est LE coeur de DenseNet. On concatène l'entrée originale de cette couche avec ses 32 nouvelles features. Le tenseur résultant devient l'entrée de la couche suivante.

Entrée couche i : (B, 64 + (i-1)×32, H, W)
Sortie couche i : (B, 64 + i×32, H, W) ← +32 canaux à chaque couche
Intuition : C'est comme si chaque couche "publiait" 32 nouvelles informations dans un tableau partagé. Toutes les couches suivantes peuvent lire tout le tableau. Les features de la couche 1 (bords simples) sont encore disponibles à la couche 6, sans transformation.
code Python correspondant (_DenseLayer.forward)
def forward(self, x):
    # self.layer = BN→ReLU→Conv1×1→BN→ReLU→Conv3×3
    out = self.layer(x)             # → 32 nouveaux canaux
    out = self.drop(out)            # dropout optionnel
    return torch.cat([x, out], dim=1) # ← concaténation sur les canaux

ce que "voit" chaque couche — c'est ça la connexion dense
La couche 6 ne voit pas que la couche 5. Elle voit simultanément :
• Les 64ch du stem (features brutes, bords simples de l'image)
• Les 32ch de la couche 1 (patterns élémentaires)
• Les 32ch de la couche 2 (combinaisons de patterns)
• Les 32ch de la couche 3 (structures émergentes)
• Les 32ch de la couche 4 (formes partielles)
• Les 32ch de la couche 5 (structures complexes)
→ Total : 224 canaux d'entrée, tous concaténés. Elle peut choisir ce qui lui est utile parmi tout ce qui a été appris avant elle.
ResNet : x_L = H_L(x_{L-1}) + x_{L-1} ← addition, 1 seule source
DenseNet : x_L = H_L([x_0, x_1, x_2, ..., x_{L-1}]) ← cat, TOUTES les sources
visualisation du gradient — pourquoi ça s'entraîne bien
∇

Chemins courts de gradient

Lors de la rétropropagation, le gradient de la loss doit remonter jusqu'aux premières couches. Dans un CNN classique, ce gradient passe par toutes les couches (et peut diminuer ou exploser). Dans DenseNet, le gradient a un chemin direct depuis la sortie jusqu'à chaque couche, via les concaténations. La couche 1 reçoit directement le gradient de la couche 6, sans l'atténuation des couches intermédiaires.

from scratch

Entraînement from scratch facilité

Votre code n'utilise pas de poids pré-entraînés (ImageNet). Ce bon gradient flow est donc crucial : sans lui, les premières couches apprendraient très lentement. C'est pourquoi DenseNet est recommandé dans votre projet par rapport à ResNet from scratch.

les deux rôles de la transition

① Compression des canaux (θ=0.5)

La Conv1×1 réduit de moitié le nombre de canaux :

256 → 128 (θ=0.5)
512 → 256
1024 → 512

Sans compression, après 4 blocs les canaux atteindraient ~2000. Le θ force le réseau à sélectionner et compacter les features les plus importantes avant de passer au bloc suivant.

out_ch = ⌊ in_ch × θ ⌋ = ⌊256 × 0.5⌋ = 128

② Downsampling spatial (AvgPool 2×2)

L'AvgPool2×2 divise la résolution spatiale par 2 :

56×56 → 28×28 (T1)
28×28 → 14×14 (T2)
14×14 → 7×7 (T3)

Pourquoi AvgPool et pas MaxPool ? L'AvgPool est plus doux : il moyenne les activations au lieu de prendre le max. Adapté ici car on ne veut pas "perdre" des features, juste les agréger doucement.

⑦ La Grille Spatiale — sortie vers l'attention

Comment (B, 1024, 7, 7) devient (B, 49, 512) et pourquoi c'est parfait pour l'attention

les 4 opérations finales
BN

BN + ReLU final (self.final_norm)

Recommandé par l'article DenseNet original. La sortie du Dense Block 4 est une concaténation de features qui n'ont pas encore été normalisées ensemble. Ce BN final normalise les 1024 canaux avant la projection.

(B, 1024, 7, 7) → (B, 1024, 7, 7)
Pool

AdaptiveAvgPool2d(7, 7)

Garantit une grille exactement 7×7 quelle que soit la résolution d'entrée. Si l'image est 224×224, après le réseau on obtient déjà 7×7 — le pool ne fait rien. Mais si on passe une image de taille différente (256×256 par exemple), ce pool corrige automatiquement.

(B, 1024, H, W) → (B, 1024, 7, 7) ← toujours 7×7
↔

permute(0, 2, 3, 1) + reshape — la transformation clé

On réorganise le tenseur pour que les 49 positions spatiales deviennent des "tokens" distincts, et les 1024 features deviennent la "valeur" de chaque token. C'est la même logique que les patches dans un Vision Transformer.

(B, 1024, 7, 7)
→ permute(0,2,3,1) → (B, 7, 7, 1024)
→ reshape(B, 49, 1024) → (B, 49, 1024)
La grille 7×7 → 49 patches (chacun = une région ~32×32px de l'image originale)

patch[48] : (1024,) — coin bas-droite
49 vecteurs de features
fc

Linear(1024 → feature_dim=512) — projection pixelwise

La même transformation linéaire est appliquée indépendamment à chacun des 49 patches. On passe de 1024 dimensions à 512 (= hidden_dim du decoder). C'est "pixelwise" car la projection est identique pour chaque position spatiale.

(B, 49, 1024) → Linear(1024, 512) → (B, 49, 512)

C'est la sortie finale de l'encodeur.
Elle est passée directement à BahdanauAttention dans le decoder.
Pourquoi feature_dim = hidden_dim du decoder ? L'attention Bahdanau projette les features (W_enc · f_i) et l'état caché (W_dec · h_t) dans le même espace (attention_dim=256). Pour que W_enc et W_dec aient des dimensions raisonnables, on aligne d'abord feature_dim avec hidden_dim via cette projection.
⑧ Pourquoi DenseNet pour le captioning ?

La synergie entre connexions denses et attention de Bahdanau

le problème des CNNs classiques pour le captioning

❌ CNN classique (VGG, ResNet léger)

1

Features de couche 10 transformées 10 fois → les textures simples "disparaissent"

2

Le gradient de la loss atteint difficilement les premières couches (from scratch)

3

La grille finale contient principalement des features de haut niveau — l'attention manque les détails fins

✅ DenseNet

1

Features de la couche 1 accessibles directement à la couche 100 via concaténation

2

Gradient court-circuite les couches → chaque couche reçoit directement le signal d'erreur

3

La grille 7×7 agrège features basse ET haute fréquence → attention plus informative

la synergie avec l'attention Bahdanau
🐕

Générer le mot "dog"

L'attention regarde le patch correspondant à la région du chien. Dans ce patch, le vecteur de 512 features contient :

  • Des features DenseNet profondes : silhouette animale, pelage, posture
  • Des features peu profondes (concaténées) : textures du pelage, contours de l'oreille

L'attention peut pondérer les features qui lui sont les plus utiles pour décider entre "dog", "cat", "wolf".

🌿

Générer le mot "grass"

L'attention se déplace vers le bas de l'image (patches du bas). Ces patches contiennent des features de texture répétitive (herbe) facilement capturées par les couches précoces de DenseNet, qui sont encore présentes dans la grille finale.

📊

Résultat mesuré

Sur COCO avec attention LSTM, DenseNet from scratch surpasse les mini-ResNets de profondeur comparable en termes de BLEU-4 et CIDEr. C'est pourquoi encoder_type='densenet' est le défaut recommandé dans votre create_model().

Rappel : tout le réseau est entraîné from scratch — pas de poids ImageNet. Le gradient flow de DenseNet est la raison principale pour laquelle ça converge correctement.
